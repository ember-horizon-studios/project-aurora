# Values

*Level 1 — The Foundation. Everything in the system is evaluated against this layer. No goal, project, or task can be principally prioritized without it.*

**Status: INCOMPLETE — define before proceeding**

A value in this system is a demonstrated pattern of choice: what you have consistently protected even when it was costly. Excavate from actual behavior, not aspiration.

---

## Excavation Exercise

Answer these before filling the table:

- What have you said no to in the last 6 months that was genuinely hard to decline? 
- What have you spent money or time on that wasn't strictly necessary but felt non-negotiable? Learning new skills in development and AI.
- What would you keep doing even if no one noticed or paid you? I would keep searching for answers.
- Looking at your calendar — what got protected even when things got busy? My free time is always protected. I don't want my life to run me.
- What parts of this studio project feel energizing vs. obligatory? Why? The newness feel energizing because it is a decision to take my life and career in a different direction.

## Values Table

| Rank | Name                 | Behavioral Definition                                                                                                               |

## What Makes These Work (or Fail)

A good value definition passes this test: you can point to a recent decision that demonstrates it. If you can't, it's an intention, not a value yet.

**Failure mode 1 — Too broad:** "Creativity" aligns with everything and eliminates nothing. The filter has no teeth.

**Failure mode 2 — Aspirational:** "I value discipline" — but you haven't demonstrated it under cost. The system will produce plans that feel right and fail in execution.

**Maintenance rule:** Review quarterly at most. When a value changes, every goal, project, and task tagged to it must be re-evaluated. That cascade cost is intentional — it keeps the foundation honest.
